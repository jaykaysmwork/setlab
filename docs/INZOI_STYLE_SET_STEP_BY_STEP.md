# inZOI 류 **도시·거리 3D 세트** 만들기 — 순서대로 상세 가이드

inZOI 같은 화면은 **“한 도구 한 번”**이 아니라 **기획 → 블로킹 → 모듈 세트 → 머티리얼·라이팅 → 소품 → (선택) 캐릭터**가 쌓인 결과입니다.  
아래는 **처음부터 끝까지** 따라갈 수 있게 **순서**를 고정한 것입니다. (완전 동일한 퀄리티·규모는 스튜디오·기간이 필요하다는 점은 전제로 둡니다.)

---

## 전체 로드맵 (한 눈에)

```
① 방향·레퍼런스 고정
② Unreal 프로젝트 준비 (도시/실외에 맞는 설정)
③ “한 블록” 범위 정하기 (온 도시 X → 길 한 줄·교차로 한 개부터)
④ 브리프 → AI 레이아웃 (이 레포 setlab) → glTF
⑤ 언리얼에 Import Into Level 로 블로킹 입성
⑥ 도로·인도 (스플라인 또는 모듈 타일)
⑦ 건물 파사드 (모듈 키트 + 반복)
⑧ 소품·가로등·표지판 (인스턴싱)
⑨ 식생·나무 (Foliage 또는 스캐터)
⑩ 하늘·안개·시간대·포스트 (분위기 = inZOI 느낌의 대부분)
⑪ (선택) 캐릭터·차량
⑫ 트레일러면: Level Sequence + MRQ
```

---

## ① 방향·레퍼런스 고정 (반나절~1일)

**목표:** “무슨 동네인지” 한 문장으로 말할 수 있게 하기.

1. **한 줄 컨셉** 적기  
   - 예: *한국 도시의 저녁 골목, 비 온 뒤 젖은 아스팔트, 4층 이하 상가*  
2. **레퍼런스 이미지 5~10장** 모으기 (Pinterest, 스트리트뷰 캡처 등) — **색·밀도·건축 언어**만 맞아도 성공 확률이 올라감.  
3. **금지 목록** 적기 (예: *고층 타워 없음, 넓은 광장 없음*) — AI 브리프에도 넣기 좋음.

**산출물:** `my_brief.txt` + 이미지 폴더 (선택)

---

## ② Unreal 프로젝트 준비

**목표:** 나중에 후회 안 하는 기본 설정.

1. Epic Launcher → **새 프로젝트**  
2. 템플릿: **Blank** 또는 **Open World** 계열이 있으면 우선 검토 (없으면 Blank).  
3. **DMX 전용 템플릿은 사용하지 않기** (조명 무대용이라 거리 세트와 맞지 않음).  
4. 프로젝트 생성 후:  
   - **Edit → Project Settings → Maps & Modes** 에서 기본 맵 저장  
   - **World Partition** 사용 여부는 “큰 맵”을 엄청 넓게 열 거면 검토 (처음엔 **비활성/작은 레벨**도 충분)

5. **플러그인**  
   - **Interchange / glTF** (이미 쓰던 대로)  
   - (선택) **Water**, **Landmass** — 물/지형이 필요할 때만 
   - (선택) **Niagara** — 나중에 비·먼지 등

**산출물:** 빈 레벨 하나 저장 (`Maps/Street_Block_01`)

---

## ③ 범위 줄이기 (매우 중요)

**목표:** inZOI 전체가 아니라 **“한 컷에 들어가는 블록”** 만 만든다.

1. **도로 길이** 숫자로 정하기 (예: **80m × 40m** 구역).  
2. **카메라가 보는 각도** 1~2개만 정하기 (예: *길을 따라 정면*, *교차로 대각선*).  
3. 보이지 않는 뒤쪽은 **박스로 막거나** 안 만들어도 됨.

**산출물:** 종이/메모에 스케치 또는 스크린샷에 박스 그리기

---

## ④ 브리프 → AI 레이아웃 (이 레포)

**목표:** 큐브 블로킹으로 **보도·차도·건물 덩어리** 위치를 잡는다.

1. `my_brief.txt` 에 ①의 컨셉 + ③의 치수 + 금지 목록을 넣는다.  
2. 터미널:

```bash
cd /path/to/setlab
source .venv/bin/activate
python -m setlab.run my_brief.txt --out out/inzoi_block --backend ollama --model llama3.2
```

3. `set_spec.json` 을 연다 — **모듈 이름·개수**가 브리프와 말이 되는지 본다.  
4. 안 맞으면 **브리프를 더 구체화**하거나 `setlab/prompts.py` 의 규칙을 조정하고 **다시 생성**.

**한계 (솔직히):** 지금 파일럿은 **복도형 프롬프트**에 가깝게 튜닝돼 있을 수 있어, **도시 블록**을 원하면 브리프에 *“도로 중앙선, 보도 2m, 블록 단위 건물 슬롯”* 을 **명시**하는 게 좋다. (필요하면 프롬프트를 도시용으로 바꾸는 게 다음 개선 과제.)

**산출물:** `out/inzoi_block/set.gltf` + `set_spec.json`

---

## ⑤ Unreal에 **한 번에** 올리기

**목표:** 손으로 조각 드래그 지옥을 피한다.

1. **File → Import Into Level…** → `set.gltf`  
2. 스케일 이상하면 Interchange에서 **단위** 조정 (미터 기준 프로젝트면 1m = 100uu 등 프로젝트 규칙에 맞춤).  
3. **Save All**

자세한 대안: `docs/UNREAL_AUTO_PLACEMENT.md`

**산출물:** 레벨 안에 블로킹 메시가 보임

---

## ⑥ 도로·인도 (거리 느낌의 뼈대)

**목표:** 회색 큐브가 **“길”** 로 읽히게.

1. **도로 폭·차선** 정하기 (블로킹 박스에 맞춤).  
2. 방법 A: **모듈러 도로 타일** (마켓플레이스/메가스캔/직접 모델링)을 **스냅**으로 깐다.  
3. 방법 B: **Landscape + 데칼** 로 아스팔트 느낌 (초보에게는 A가 더 쉬운 경우 많음).  
4. **보도 블록**을 도로보다 **살짝 높이** (Z 오프셋) — 현실감에 도움.

**산출물:** 도로 메시가 블로킹을 덮는 상태

---

## ⑦ 건물 파사드 (inZOI 느낌의 대부분)

**목표:** **같은 창·벽 모듈**을 반복해 “동네”처럼 보이게.

1. **1층 상가 / 2~4층 주거** 처럼 **층고 규칙** 정하기.  
2. **Facade 모듈 세트** (창문 패널, 기둥, 코너) 준비 — Megascans / Fab / 자체 모델.  
3. 블로킹 박스 **크기에 맞춰** 모듈을 **그리드에 스냅**해서 쌓는다.  
4. **반복을 줄이려면**: 창 위치만 다른 **배리언트** 머티리얼 2~3개.

**산출물:** 거리 양쪽에 “건물 벽”이 읽힘

---

## ⑧ 소품·가로등·표지판

**목표:** 삶의 흔적.

1. **가로등** — 일정 간격으로 **스플라인** 또는 **간격 복제**.  
2. **표지판·에어컨 실외기·전봇대** — 작은 메시를 **인스턴스 스태틱 메시**로 찍기.  
3. **너무 많이** 넣지 않기 — **실루엣**이 먼저 읽히게.

---

## ⑨ 식생

**목표:** 길가 나무·화단.

1. **나무** 몇 그루만 — **Foliage** 모드로 페인트 또는 수동 배치.  
2. **잔디**는 나중에 — 처음엔 **도로+건물**만 잡아도 inZOI “거리” 느낌의 70%는 온다.

---

## ⑩ 하늘·안개·시간대·포스트 (분위기)

**목표:** **영상 퀄리티의 절반**은 여기서 나온다고 보면 됨.

1. **Directional Light** + **Sky Atmosphere** + **Sky Light** — 저녁/낮 정하기.  
2. **Exponential Height Fog** — 거리 감.  
3. **Post Process Volume** — 대비, 색보정, 살짝의 필름 그레인(선택).  
4. **젖은 바닥** 느낌: 머티리얼에서 **러프니스/노멄** 조정 또는 **데칼**.

---

## ⑪ (선택) 캐릭터·차량

inZOI는 **사람·차**가 많다. 첫 세트에서는 **스킵**하고 **빈 거리**만 잡아도 학습 가치는 충분하다.  
넣을 경우: **메타휴먼** 또는 단순 **스태틱 메시** 실루엣으로 멀리 서 있게.

---

## ⑫ 트레일러 (30초)

1. **Level Sequence** — 0~30초.  
2. **Cine Camera** — ③에서 정한 1~2 앵글 위주.  
3. **Movie Render Queue** — ProRes/시퀀스 → `.mov`.

`docs/WALKTHROUGH_MOV_PIPELINE.md` Phase 6~8 참고.

## 당신의 파일럿(`setlab`)이 끼는 위치

- **④~⑤** 사이: **빠른 블로킹·반복**  
- **⑥~⑩**: **아트·모듈·라이팅** (시간의 대부분)  
- inZOI급 **최종 화면**은 **⑥ 이후**에서 결정된다고 보면 된다.

## 다음에 손대면 좋은 “코드/프롬프트” 개선 (선택)

- `setlab/prompts.py` 를 **도시 블록 전용** (도로/보도/슬롯)으로 바꾸기  
- `set_spec.json` 의 `asset` 이름을 **실제 모듈 ID**와 1:1 매핑표로 연결

이 두 가지를 하면 **AI → 언리얼** 루프가 inZOI 스타일에 더 가까워진다.
